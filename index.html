What is Pytest?

Pytest is Python's most popular testing framework. It replaces the verbose unittest style with clean, simple functions. No boilerplate classes required — just write functions that start with test_.

OLD WAY — unittest
import unittest

class TestMath(unittest.TestCase):
    def test_add(self):
        self.assertEqual(1 + 1, 2)

if __name__ == '__main__':
    unittest.main()
NEW WAY — pytest ✓
def test_add():
    assert 1 + 1 == 2


# That's it. Run: pytest

Why pytest wins: Auto-discovery of tests, powerful fixtures, parametrize, rich assertion introspection (it shows you what failed, not just that it failed), 1000+ plugins, and it's the industry standard.

Step 3 — Project layout

Pytest auto-discovers tests. Just follow the naming convention:

my_project/
├── my_module.py          # Your source code
├── conftest.py           # Shared fixtures (auto-loaded)
├── test_my_module.py     # Tests (prefix: test_)
└── pyproject.toml        # Configuration

Naming rules: Files must start with test_ or end with _test.py. Functions must start with test_. Classes must start with Test.

Your First Test

test_example.py
# 1. A simple function test
def test_addition():
    assert 1 + 1 == 2

# 2. Testing for exceptions
import pytest

def test_division_by_zero():

        1 / 0

class TestStringMethods:
    def test_upper(self):

    def test_startswith(self):

Core Concepts

01 assert — The Foundation

Pytest uses plain Python assert statements. When they fail, pytest shows exactly what went wrong.

assert result.success is True          # Boolean check
assert "error" in result.message       # Membership
assert len(items) == 5                 # Equality
assert result is not None              # Identity

02 pytest.raises — Testing Exceptions

Verify that code raises the expected exception. The test passes only if the exception is thrown.

with pytest.raises(TypeError):
    DocumentTask(document_id="d1")  # Missing required fields

with pytest.raises(SystemExit) as exc_info:
    await run_worker()
assert exc_info.value.code == 1  # Check exit code

03 Markers — Labeling Tests

Markers tag tests with metadata. The most common is @pytest.mark.asyncio for async tests.

@pytest.mark.asyncio
async def test_async_function():
    result = await some_async_call()
    assert result == expected

@pytest.mark.skip(reason="Not implemented yet")
def test_future_feature():
    pass

@pytest.mark.parametrize("input,expected", [(1, 2), (3, 4)])
def test_increment(input, expected):
    assert input + 1 == expected

04 Test Classes — Grouping Related Tests

Group related tests in classes for organization. No inheritance needed — just prefix with Test.

class TestExtractCsv:
    def test_csv_extraction(self, tmp_csv_file):
        ...
    def test_tsv_extraction(self, tmp_tsv_file):
        ...
    def test_large_csv_truncates(self, tmp_path):
        ...

Fixtures

Fixtures are pytest's way of providing setup and teardown. A test declares what it needs by naming fixtures in its function signature — pytest injects them automatically.

Built-in Fixtures

Fixture What it does
tmp_pathGives a unique temporary directory (pathlib.Path) per test
tmp_path_factorySession-scoped temp directory factory
capsysCaptures stdout/stderr
monkeypatchModify/replace objects, env vars, paths
requestAccess to the requesting test context

Custom Fixtures (from our conftest.py)

# conftest.py — auto-loaded by pytest, no imports needed

@pytest.fixture
def sample_task():
    """Factory fixture — returns a function that creates DocumentTask."""
    def _make(document_id="doc-001", filename="report.pdf", **kw):
        return DocumentTask(document_id=document_id, filename=filename, ...)
    return _make

@pytest.fixture
def tmp_csv_file(tmp_path):  # ← uses built-in tmp_path!
    """Creates a temp CSV file for tests."""
    p = tmp_path / "data.csv"
    p.write_text("name,age\nAlice,30\nBob,25\n")
    return p

@pytest.fixture
def mock_activity_heartbeat():
    """Patches Temporal heartbeat so tests run without a server."""
    with patch("temporalio.activity.heartbeat") as mock_hb:
        yield mock_hb  # ← yield = teardown happens after test

Using Fixtures in Tests

# Just name the fixture in the function signature — pytest injects it

async def test_csv_extraction(self, tmp_csv_file):
    result = _extract_csv(tmp_csv_file)
    assert "Alice | 30" in result

async def test_file_not_found(self, sample_task, mock_activity_heartbeat):
    task = sample_task(file_path="/nonexistent/file.pdf")
    result = await extract_text(task)
    assert result.success is False

Fixture scopes: @pytest.fixture(scope="function") (default — runs per test), scope="class", scope="module", scope="session". Use autouse=True to apply to all tests in scope. Docs ↗

Mocking & Patching

Mocking replaces real objects with fake ones so you can test in isolation — without a database, network, or external service.

Key Tools

Tool When to use
MagicMock()Create a fake object that records calls. Accepts any attribute/method.
AsyncMock()Like MagicMock but for async functions — returns a coroutine.
patch("module.object")Temporarily replace an object at its import path. Auto-restores after test.
patch.dict("sys.modules", {...})Inject mock modules for libraries imported inside functions (e.g. import docx).

Pattern 1 — Patching a module-level import

# activities.py imports Elasticsearch at module level
# We patch where it's USED, not where it's defined

with patch("activities.Elasticsearch", return_value=mock_es):
    result = await index_to_elasticsearch(task, extraction)

mock_es.index.assert_called_once()  # Verify it was called

Pattern 2 — Patching a local import (sys.modules)

# worker.py does `import docx` INSIDE _extract_docx()
# Can't use patch("worker.docx") — it doesn't exist at module level
# Solution: inject a mock module into sys.modules

mock_docx = MagicMock()
mock_docx.Document.return_value = mock_doc_object

with patch.dict("sys.modules", {"docx": mock_docx}):
    result = _extract_docx(path)  # Now `import docx` finds our mock

Pattern 3 — Mocking async HTTP clients

# The code uses: async with httpx.AsyncClient() as client:
# We mock the context manager (__aenter__ / __aexit__)

mock_response = MagicMock(status_code=200)
mock_client = AsyncMock()
mock_client.post.return_value = mock_response

with patch("activities.httpx.AsyncClient") as MockClient:
    MockClient.return_value.__aenter__ = AsyncMock(return_value=mock_client)
    MockClient.return_value.__aexit__ = AsyncMock(return_value=False)
    result = await update_document_status("doc-001", "completed", "text")

assert result is True
mock_client.post.assert_called_once()

Async Testing

The pytest-asyncio plugin lets you write async def test_... functions. It manages the event loop for you.

pyproject.toml
[tool.pytest.ini_options]
asyncio_mode = "auto"  # ← All async tests run automatically, no decorator needed
test_activities.py
# With asyncio_mode = "auto", just write async tests normally:

async def test_successful_extraction(self, sample_task, tmp_text_file):
    task = sample_task(filename="sample.txt", file_path=str(tmp_text_file))
    with patch("activities._extract_with_unstructured", return_value="content"):
        result = await extract_text(task)
    assert result.success is True

Without auto mode: Add @pytest.mark.asyncio above each async test function. Docs ↗

Code Coverage

pytest-cov measures which lines of your code are executed during tests. It wraps the coverage.py library. 

# Run with coverage for specific modules
pytest --cov=shared --cov=worker --cov-report=term-missing

# Branch coverage (tests if/else paths, not just lines)
pytest --cov=worker --cov-branch

# Fail if coverage drops below threshold
pytest --cov=worker --cov-fail-under=90

# Generate HTML report you can browse
pytest --cov=worker --cov-report=html:htmlcov

Configuration

Put all pytest settings in pyproject.toml — one file for everything.

pyproject.toml (from our project)
[tool.pytest.ini_options]
asyncio_mode = "auto"                # Auto-detect async tests
testpaths = ["."]                    # Where to look for tests
python_files = ["test_*.py"]         # File discovery pattern
python_classes = ["Test*"]           # Class discovery pattern
python_functions = ["test_*"]        # Function discovery pattern
addopts = ["-v", "--tb=short"]       # Default CLI options

[tool.coverage.run]
source = ["shared", "activities", "workflows", "worker"]
branch = true                        # Enable branch coverage
omit = ["test_*", "conftest.py"]     # Exclude test files

[tool.coverage.report]
show_missing = true
fail_under = 90                      # Enforce minimum 90%
exclude_lines = [
    "if __name__ == .__main__.",     # Skip entry-point guard
    "pragma: no cover",              # Explicit skip marker
]

PART 3 — REFERENCE

Best Practices

1. One assert per concept

Each test should verify one behavior. Multiple asserts are fine if they validate the same concept.

2. Name tests descriptively

test_file_not_found_returns_failure tells you exactly what it tests, even without reading the code.

3. Use fixtures for setup, not copy-paste

Put shared setup in conftest.py. Factory fixtures (functions that create objects) are more flexible than static ones.

4. Mock at the boundary, not inside

Mock external services (HTTP, database, file system), not your own code. Test your logic, fake their responses.

5. Patch where it's used, not where it's defined

patch("activities.Elasticsearch") not patch("elasticsearch.Elasticsearch"). Patch the reference the code actually uses.

6. Test the sad paths

Happy-path tests aren't enough. Test failures, exceptions, edge cases, empty inputs, and timeouts.

7. Use branch coverage

--cov-branch catches untested if/else paths that line coverage misses. Aim for 90%+.

8. Keep tests fast

Mock I/O, avoid real network calls, use tmp_path instead of real filesystems. Our 86 tests run in ~8 seconds.

Cheat Sheet

## ASSERTIONS
assert x == y                        # Equality
assert x != y                        # Inequality
assert x is True                     # Identity
assert x is not None                 # Not None
assert "text" in string              # Substring
assert item in list                  # Membership
assert len(x) == 5                   # Length

## EXCEPTIONS
with pytest.raises(ValueError):      # Expect exception
with pytest.raises(SystemExit) as e: # Capture exception info
    assert e.value.code == 1

## FIXTURES
@pytest.fixture                      # Define fixture
@pytest.fixture(autouse=True)        # Auto-apply to all tests
@pytest.fixture(scope="session")     # Run once per session
def my_fixture():
    setup()
    yield value                      # yield = teardown after
    teardown()

## MARKERS
@pytest.mark.asyncio                 # Async test
@pytest.mark.skip(reason="...")      # Skip test
@pytest.mark.skipif(cond, reason="") # Conditional skip
@pytest.mark.parametrize("a,b", ...) # Run with multiple inputs

## MOCKING
MagicMock()                          # Fake object
AsyncMock()                          # Fake async object
patch("module.thing")                # Replace for duration
patch.dict("sys.modules", {...})     # Inject mock modules
mock.assert_called_once()            # Verify call
mock.call_args.kwargs["key"]         # Inspect arguments

## CLI
pytest -v                            # Verbose output
pytest -x                            # Stop on first failure
pytest -s                            # Show print output
pytest -k "keyword"                  # Filter by name
pytest --tb=short                    # Short tracebacks
pytest --cov=module                  # Coverage report
pytest --cov-branch                  # Branch coverage
